# 建议的精简重构路线

如果你在前端联调后准备再整理一版后端，我建议按这个顺序做。

## 第 1 批：不影响协议的小清理
- 打包时排除 `__pycache__`
- 给 Python 层补 `__init__.py`
- 把 README 与实际命令保持完全同步
- 补一份 error code 表

## 第 2 批：不改功能，只提稳定性
- `RobotClient` 增加状态流自动重连
- `ExperimentManager` 增加统一错误映射
- replay 样本改成共享只读引用，去掉大拷贝

## 第 3 批：仅当前端确实需要时再做
- 状态推流多客户端广播
- 命令仲裁与会话管理
- 更正式的前端服务层（如 FastAPI 本地桥接）

## 我不建议现在做的事
- 不要把控制核心搬到 Python
- 不要为了 GUI 去改底层 MotorIO 语义
- 不要一开始就上重型 Web 全栈
